Communication Troubleshooting for the eTPU

 

  • If the Yellow LED (AC Power) is not on, then there is no power present.
    To correct this, make sure the eTPU power supply is properly connected to the eTPU and connected to an AC power source. Use only a factory-supplied power supply.

 

  • If the green LED (Network communication) is not on, then there is no Ethernet communication.
    To correct this, ensure the dedicated Ethernet adapter is configured properly. Also make sure the correct IP address class is used, and that the subnet mask is entered correctly.

 

  • If you open the software and the red LED does not begin blinking (slow blink), then the SitePlayer IP address may not be configured properly.
    To correct this, exit the software and then open the file:
    C:\software root directory\Log\KIC2000Hardware.kiccfg

    Edit the line:
    SitePlayerIPAddress=

    Enter the IP address of your eTPU-SitePlayer.
    If still not working, then use Internet Explorer to access the eTPU SitePlayer and use the manual switch.

 

  • If you cannot connect to the SitePlayer using Internet Explorer, then the SitePlayer IP address is typed incorrectly or the address is unknown.
    To correct this, enter the correct IP address for the SitePlayer. As long as the SitePlayer is not damaged, you can connect. If you are unsure of the correct IP address of the SitePlayer, remove the cover and read the IP address from the eTPU configuration label. If the IP address is unrecoverable, then the eTPU must be returned to your system supplier for reprogramming.

 

  • If communication with the eTPU is lost, (i.e. eTPU Ethernet cable unplugged, or the power disconnected) the system may not recover on its own.
    To correct this, close the software and then open it again. This should reset the eTPU SitePlayer. In some situations, it may require a PC reboot to regain communication with the eTPU. (green and red LEDs)
