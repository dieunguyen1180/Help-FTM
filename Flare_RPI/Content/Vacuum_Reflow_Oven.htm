Vacuum Reflow Oven Systems And Functionality

Vacuum reflow ovens typically include an extra chamber in between the heated and cooling sections of the oven. During production, the PCB travels into the vacuum zone, the chamber is sealed off from the rest of the process tunnel, and a vacuum is drawn. The chamber is opened, and then the PCB continues into the cooling section and exits the oven. The purpose of adding a vacuum chamber to the process is to reduce voiding in the formation of the solder joints. The RPI system can be integrated into select* vacuum reflow ovens. This section outlines the unique capabilities of the RPI Vacuum system.

* Contact KIC to check for compatibility with your machine

 

In addition to the other functionality described in this manual, the RPI Vacuum system includes:

  • Process monitoring capabilities throughout the entire process, including the vacuum zone
  • Time duration PCB spent in vacuum zone automatically saved with each profile
  • Time duration PCB spent in vacuum zone automatically saved for each PCB during production
  • A dynamic graphical display of the vacuum zone for each PCB (based on the time each PCB spent in the chamber)

Note: Software communication with the applicable oven controller software may be required for the RPI Vacuum system.

RPI Vacuum System Hardware

The RPI Vacuum system incorporates custom built thermal probes unique to the vacuum ovens which allow for monitoring the heated zones, vacuum chamber, and cooling section to provide complete monitoring of the PCB through the process.

RPI Vacuum System Functionality

During profiling, the system identifies the duration that the PCB spent in the vacuum chamber, and the graphical display of the vacuum zone is drawn to match this time. A window is also displayed (and saved with the profile) showing how much time, in seconds, the PCB spent in the chamber.

On vacuum ovens, even if the vacuum chamber is not in use, the software will still display the vacuum zone on the graph, and show the time spent in that zone (based on length of zone defined and conveyor speed setpoint).

During production, as with systems installed on non-vacuum ovens, the live-mode screen displays both the baseline and virtual profile information. To distinguish between the two, the baseline profile appears in a solid line format, while the virtual profile appears in dashed lines.

Additionally, a green line (End of Vacuum Zone Indicator) appears on the graph representing the calculated end of the vacuum zone. Depending on the time a given PCB spends in the chamber, this green line will shift forward or back to show the relative ‘size’ of the zone. More time in the chamber equates to a ‘longer’ zone, and less time a ‘shorter’ zone.

Additional Warning for Vacuum Process

On RPI Vacuum systems, an additional warning can be enabled for users who want to be notified if the time that production boards spend in the vacuum chamber varies too much from their baseline profile setup.

 

To enable this feature, access the Global Preferences/Control Limits tab and select the checkbox for – ‘Trigger Warning if vacuum time varies from baseline by more than’.

 

Enter the percentage of variation in time you will allow before the system should display the warning message.

 

Additionally, you can select the Disable Alarm tab to further customize what happens if this warning occurs.

NOTE: Selecting a checkbox on this screen DISABLES the particular function.

 

By default, the warning message will appear on the screen (checkbox not selected), the Alarm Relay will NOT be activated (checkbox selected), and the warning will be included in Live Data Output files (when LDO is in use and enabled).

 

Click the green checkmark button to save any changes and return to the RPI main screen.

Additional Board Sensing Logic for Vacuum Process

On some dual lane vacuum reflow machines, stoppers may be used by the oven manufacturer to sync up the products in the two lanes before allowing them to enter into the heated zones. In these configurations, a special function can be enabled which will alter the standard board sensor operation so that if a product is stopped (and is underneath the board sensor), the board sensor will only detect it as a single product.

 

To enable this feature, shut down the software and open the C:\RPI\ConfigurationProgram.exe application.

 

On the Hardware Tab, select the Enable Fixed Board Length Logic checkbox. When this feature is enabled, you can also enable an alarm notification to alert you if the board sensor is triggered too long (board stuck, improper sensor function, etc.). Select the Alarm if Board Sensor On for “x” seconds checkbox and then enter the amount time delay before the alarm should be generated. Click Apply and OK to save your changes and exit the Configuration utility.
