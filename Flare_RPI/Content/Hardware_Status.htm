Hardware Status

 

Figure 20: Hardware Status Screen

 

Oven Controller – When connected to an oven that is capable of communicating with the RPI software, the software will display oven status and display the oven control software name.

 

Profiler – Shows the status of the profiler. This box displays the following information:

  • Profiler communication status.
  • Temperature display – for all connected thermocouples.
  • Battery information.
  • Internal Temperature – Temperature inside the profiler.
  • Maximum Temperature Capability – This value determines the maximum temperature the profiler can read.

 

Software key – Displays all of the features currently programmed on the USB Dongle key.

 

Probes – The current live temperatures for the probes are displayed. A third line displays hardware information for the Board Sensor, and Conveyor Speed Encoder. If you do not see any data displayed in this field, the RPI software is not communicating properly with the eTPU hardware.

 

    Note: If you are using the barcode capabilities, the last barcode read by the scanner also appears in the Barcode section, as well as your enabled barcode features.

 

Test eTPU alarm relay – Use this button to test the alarm relay function of the software. The alarm relay hardware must be properly connected. Use this button to manually disable the alarm, should it stay active after Virtual Profiling is not running. The button appears as long as the software is detecting the eTPU device.

 

    Note: The alarm relay is not energized until VP is first running and then goes out of spec. To learn more about this see the alarm relay section.
