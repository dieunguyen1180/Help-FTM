Part 1: The Basic System

The Hardware

There major hardware components to the basic system:

  • DAU (Data Acquisition Unit)

  • Probes

  • Conveyor speed encoder

  • Board sensor

 

DAU

The DAU interfaces with the computer using an Ethernet cross-over cable that connects from the UpLink port on the device directly to a USB to Ethernet adapter (or to an installed Network Interface Card (NIC)). Please follow manufacturers’ instructions to install the USB adapter or NIC.

 

The computer used with the WPI system can be a machine that is dedicated to only supporting WPI or it can also serve as the machine controller PC.

 

The DAU must be plugged into an AC power source. It utilizes a universal A/C transformer power supply and ships from the factory with multiple AC connectors to accommodate any connector/voltage requirement.

 

Probes

The probes are mounted in the oven process chamber. They are usually suspended from the conveyor rails by oven manufacturer approved brackets. The probes are connected to the DAU – EVEN/ODD ports, and each probe connector is stamped with an E or O in order to designate the EVEN or ODD numbered probe TCs.

 

Conveyor Speed Encoder

The conveyor speed encoder is mounted to read pulses from a gear that turns while the conveyor is operating. The conveyor speed encoder connects by default to channel #1 of the DAU Sensor ports.

 

Board Sensor

The board sensor is mounted near the entrance of the machine, typically on the fixed rail, and provides for product tracking. It is connected by default to channel #2 of the DAU Sensor ports. Dual lane systems use a separate board sensor for each lane.

 

Additional Hardware

Profiler

The profiler is the data acquisition component of the system.  It provides the on-board, through-the-process temperature data.  The software analyzes this data together with oven zone temperature and conveyor speed factors to display the thermal profile graphically.

You can use software with several profiler models such as:

  1. SPS Smart Profiler

  2. X5
  3. K2

 

The software works the same with any of the profilers. Once you have the software running, you indicate your specific profiler model. The program then displays the text and graphics that reflect the model you chose. When afterward running the program, the text and graphics that appear reflect the model you selected. (The artwork that appears throughout this manual reflects a variety of profiler models.)

 

Profilers can operate in either of two modes—datalog and wireless. In standard datalog mode, the profiler records temperature data as it moves through the thermal process. A cable then transfers the data to your software application computer. In wireless mode, the profiler transmits data directly to the computer by radio signals or Wi-Fi instead of an attached cable. The application software can then display the accumulating data in real-time.

For specific information regarding your profiler, consult one of the following publications:

  • SPS Smart Profiler Hardware Guide (Publication Number PTG-330310-000)
  • X5 Profiler Hardware Guide, (Publication Number EXP-337310-000)

  • K2 Profiler Hardware Guide, (Publication Number EXP-331310-000)

Dual Lane Systems

The automatic system can monitor production through dual lane reflow ovens. Dual lane functionality supports two control configurations:

 

  1. Dual Lane Dual System
  2. Dual Lane Single System

 

A Dual Lane-Dual System configuration uses two instances of the software application and hardware to track different recipes through the oven.

 

The Dual Lane-Single System tracks two lanes of the same recipe with one instance of the software. A separate license is required for Dual Lane Single System use.

 

See Dual Lane Systems and Functionality for details on installation and configuration.
