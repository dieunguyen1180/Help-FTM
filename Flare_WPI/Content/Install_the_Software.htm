Introducing the WPI system

The WPI (Wave Process Inspection) system is a comprehensive suite of thermal process management tools designed to give you the ultimate in PCB soldering quality control. WPI can monitor the entire wave solder process, giving advanced warning before a product drifts out of specification.

 

WPI combines software and hardware components into an integrated system. The software supplements its thermal profile analysis core with special tools for performance optimization, changed variable prediction, and data file management. The WPI hardware utilizes an array of sensors that measure process temperatures, sensors to track product movement and conveyor speed, a profiler (not included) to move through the process accumulating thermal data, an Ethernet communication hub, and alarm system.

The hardware

The major hardware components to a WPI system are:

  • DAU (Data Acquisition Unit)
  • Probes – 30 thermocouples
  • Conveyor speed encoder
  • Board sensor
  • Fixed Pyrometer sensor

  • Alarm Relay/Light Tower

DAU

The DAU interfaces with the computer using an Ethernet cross-over cable that connects from the UpLink port on the device directly to a USB to Ethernet adapter (or to an installed Network Interface Card (NIC)). Please follow manufacturers’ instructions to install the USB adapter or NIC.

 

The computer used with the WPI system can be a machine that is dedicated to only supporting WPI or it can also serve as the machine controller PC.

 

The DAU must be plugged into an AC power source. It utilizes a universal A/C transformer power supply and ships from the factory with multiple AC connectors to accommodate any connector/voltage requirement.

Probes

The probes are mounted in the machine process chamber. They are usually suspended above and below the product. The probes are connected to the DAU – EVEN/ODD ports.

Each probe connector is stamped with an E or O in order to designate the EVEN or ODD numbered probe TCs. See the Hardware diagram.

Conveyor Speed Encoder

The conveyor speed encoder is mounted to read pulses from a gear that turns while the conveyor is operating. The conveyor speed encoder connects by default to channel #1 of the DAU Sensor ports.

Board Sensor

The board sensor is mounted near the entrance of the machine and provides for product tracking. It is connected by default to channel #2 of the DAU Sensor ports.

Fixed Pyrometer Sensor

The fixed pyrometer sensor is mounted outside the process tunnel, at the exit of the machine to read the direct temperature of each board as they exit the machine. It connects by default to channel #4 of the DAU Sensor ports, or through an adapter module connected to a PC USB port when used in conjuction with an older eTPU.

Alarm Relay

The Alarm Relay connects to the Alarm/Barcode port of the DAU and allows for integration to an infeed conveyor (available with SMEMA interface connections) to prevent products from entering machine when an alarm condition occurs or hardwiring to an external device of your choosing. Additionally, it allows for a standard KIC supplied audible light tower to be powered and triggered. The Alarm Relay is required to have an external AC power source.

Light Tower

There are two audible light towers available – one model connects directly to the Alarm Relay device; the other model connects directly to the USB port of the PC.

Additional Hardware

Profiler

The profiler is the product specific data acquisition component used in conjunction with the WPI system. It provides the on-board, through-the-process temperature data. WPI analyzes this data together with machine temperatures and conveyor speed factors to graphically display the thermal profile. The following profiler models are compatible with the system:

 

  1. SPS Smart Profiler
  2. X5
  3. K2

 

The software works the same with any of the profilers. Once you have the software running, you indicate your specific profiler model. (The artwork that appears throughout this manual reflects a variety of profiler models.)

 

Certain profilers can operate in either of two modes—datalog and wireless. In standard datalog mode, the profiler records temperature data as it moves through the thermal process. A cable then transfers the data to the WPI software computer. In wireless mode, the profiler transmits data directly to the computer by radio signals or Wi-Fi instead of an attached cable. WPI software can then display the accumulating data in real-time.

 

For specific information regarding your profiler, consult one of the following KIC publications:

  • SPS Smart Profiler Hardware Guide, (Publication Number PTG-330310-000)
  • X5 Profiler Hardware Guide, (Publication Number EXP-337310-000)
  • K2 Profiler Hardware Guide, (Publication Number EXP-331310-000)

Barcode Reader

A user-supplied, serial port barcode reader can connect through the DAU Alarm/Barcode port. When both the Alarm Relay and a serial port barcode reader are used, the barcode reader hardware can be connected using the Alarm Relay Y- cable. Alternately, a USB barcode reader can connect through one of the DAUs built-in USB ports. For additional information about using barcode scanners, see Appendix B – Barcode Tab.

Install The Software

Minimum PC system requirements

 

  • 2 GHz processor PC with 4GB RAM

  • 10 GB available storage

  • Video 1024 x 768 / 16-bit color

  • 1 available USB port (for Profiler)

  • 1 available USB port (for Dongle key)

  • 1 available USB port (for Light Tower)

  • 1 available Ethernet port or 1 available USB port with Ethernet-to-USB adapter

  • For product compatibility with Windows® operating systems, visit our website at: http://kicthermal.com/support-download/os-compatibility-chart
    1. When the software is running on the same computer as the machine controller, and/or when additional options are installed, a faster CPU, more RAM, and/or USB hub may be required.
    2. Solid State Disk drive and at least 8 GB RAM is recommended for better performance.


Notes Before Installation

  1. Shut down all other applications as a precaution against software conflicts. 
  1. SlimKIC 2000 profiler users should disable or uninstall all applications that interface with the PC through a serial COM port, other than your oven controller software. For example: PDA communication software.
  • If you are installing the WPI software on an oven that is capable of communication with the software, DO NOT place the WPI icon in the Windows Startup folder. If the WPI software loads before the oven controller software, it can cause the oven controller software to malfunction.

 

 

Languages

The software supports the following languages:

  • English
  • Japanese
  • German
  • Korean
  • Spanish
  • Simplified Chinese
  • Traditional Chinese

 

The use of specific languages depends on the operating system:

  • English, German, and Spanish can be run on any of the supported Windows operating systems.
  • Japanese, Korean, Simplified Chinese, and Traditional Chinese cannot be used with English versions of Windows operating systems.
  • To run the software in Japanese, the operating system must be Windows-Japanese version.
  • To run the software in Simplified or Traditional Chinese, the operating system must be Windows-Chinese version.

Production Mode - If you wish to run profiles or Virtual Profile, enter the current conveyor speed for your machine and choose the OK button, this will activate Production mode. Entering the current machine speed allows it to identify/position products that may enter the machine before the monitoring for a given product is started.

Note: To view historical Virtual Profile data while Virtual Profiling live, the user will need to launch a separate KIC Host.exe file located in: C:\WPI\AppforViewer\KICHost.exe. This will open a separate Profile Explorer that lets you browse through the history of any product including the one currently running VP. Historical data will be available for any boards that have already exited the machine.

View History Mode - If you wish to work offline and do not plan to profile or Virtual Profile, choose I am not going to run profiles or live WPI button; this will activate History mode. While History mode is active, you cannot run profiles or Virtual Profile, as the software will not communicate with the WPI hardware. The software will not even check for KIC hardware unless Production mode is active. If working in History mode you can browse to any working directory in Profile Explorer, even over a network.
