How to avoid a Grey Crystal Ball

A green crystal ball reports the system as VP-capable, yellow/red alert indicates problems, and grey indicates that the software has insufficient data to make useful VP predictions. The condition can occur during a verification profile run where the user has introduced the profiler into an ongoing production flow. With boards both downstream and upstream of the profile board, the system lacks necessary data obtained by sampling empty machine conditions. To avoid the grey status KIC recommends running the profile with no boards following behind. For more information, see Integrating Empty Oven Data.

Verify the Virtual Profile

The virtual profile is a prediction of the process profile (inside the machine) generated from a model created when the Baseline profile was run. A verification profile is a comparison run against the Virtual Profile model. If the user thinks the process has changed significantly or has purposely changed the recipe, running a verification profile lets the user verify the prediction model. This comparison can either verify that the process profile is still within specification or create a new baseline profile based on the recent changes.

 

The verification profile updates the Baseline profile for the current Virtual Profile, ensuring that its prediction model yields the most accurate possible results, using the most recent profile. The software always recognizes the latest in spec profile as the Baseline for any Virtual Profile.

 

The software recognizes when the process temperatures change significantly as compared to the current Baseline profile, and then prompts to run a verification profile. The user can intentionally run a verification profile at any time during Virtual Profiling Live Mode.

Barcode Functions

WPI software has two main barcode functions, Process Traceability, and Process Control. For either barcode function, a serial barcode scanner would connect directly to the DAU Alarm/Barcode port or to the pigtail connector of the alarm relay-cable if an alarm relay is also being used. The alarm relay is required for use of the Process Control barcode function. Alternatively, a USB scanner can also be connected directly to the DAU through one of the USB ports. See Appendix B – Barcode for more information.

 

Process Traceability (PT)

The Process Traceability function enables process traceability with profile data for every product linked to its barcode number. The software will match the barcode number and complete process data to every single production board recorded by the system. The Process Traceability Barcode function can be used with or without the Process Control function.

Process Control (PC)

The Process Control function enables process control by reading the product barcode label before the product enters the machine and confirms that a specific area of the barcode correlates to a specified product. If the barcode string value matches the current process string value, production continues as normal. If the incoming barcode string value differs from user defined string value, an alarm is activated, and the process can be stopped until the operator acknowledges the alarm and takes the proper action. In order to utilize the Process Control feature, the barcode strings need to contain at least one character (referred to as a Control String or Product ID) that is common to all boards in a production run. For example, when running “Product A”, all barcode labels for Product A would be “ABCxxxx”, where “ABC” would represent the common characters (control string) on each barcode label, and “xxxx” would represent the rest of the barcode characters, which would typically be unique to each individual board. The common character string can fall anywhere within the barcode. The Process Control Barcode function can be used with or without the Process Traceability function.

Barcode Product Sensor

The Barcode Product Sensor (BPS) is an available product sensor used exclusively with the barcode option for added functionality. Using a BPS, the WPI system can determine if any given product has had its barcode label successfully scanned. If the product passes the sensor and there are no barcode strings that have been read into the software, then the system will give a NO READ alarm.

Scanner settings

The WPI barcode function is offered as a software only feature. The user must provide the serial port or USB barcode scanner, support/ utility software, all communication cabling, and mounting hardware. Here are some important points to consider when planning to use WPI with the barcode option:

  • The barcode reader must have either a 9-pin D-Sub serial connector (RS-232) for communication or a USB model that can be programmable for Serial Port interface.

  • Barcode readers with Raster scan (multi-scan) are recommended.

  • Barcode reader must be capable of the following communication settings:

    Baud Rate: 9600 bits/sec, Data Bits: 8, Parity: None, Stop Bits: 1

 

  • The scanner(s) must be programmed to have both a “Line Feed” and “Carriage Return” after each string is scanned.

  • The scanner must be programmed for a single output for each board scanned.

  • The user will need the barcode reader utility software to configure the barcode reader.

  • Non-standard cabling may be required for multiple barcode applications.

  • When using multiple barcode scanners, the use of prefixes is required.

Barcode Types

Since the software does not actually read the barcode labels, the user can use any barcode type that the barcode scanner is capable of reading. Verify the barcode scanner can read the barcode type prior to KIC installation. Use the manufacture’s utility, or Windows-HyperTerminal for verification.

There are two methods for defining the characters of a barcode string that will be used for Process Control – Wildcard (default) and Parsing. See Appendix B – Barcode for how to change between methods.

 

Wildcard - When Virtual Profiling is started, the system produces the Barcode Process Control dialog box. This dialog box gives the opportunity to: Enter the Process Control String, Change the Process Control String, or Disable Process Control for this Virtual Profile run. If the Process Control function is enabled in Global Preferences and the Wildcard method is selected (Configuration Program), this dialog box will appear every time Virtual Profiling is enabled. Enter the barcode control string that is specific to the product family that is running. The software reads the incoming string and compares it to that of the specified Process Control String, for every product. If the string values differ, an alarm is activated, and the process is stopped via the alarm relay device.

 

Figure 112: Barcode Process Control dialog

 

Parsing – Whereas with the Wildcard method, the Control String characters must be sequential, using the Parsing method allows the user to specify characters in any position of the string to be the Control String.

In Profile Explorer, right click in the Control String column for a given product and choose Add Control String.

Figure 113: Add Control String

 

 

A window will open for selecting defining the characters of the control string:

Figure 114: Add Control String tab

 

Scan a barcode for the selected product and the full string will be displayed in the window. The user can then select the individual characters that will be the Control String for this product family. Click Add to save this Control String to the product.

Figure 115: Scan Barcode to add Control String

 

The Control String column will now display the defined string for that product name:

Figure 116: Profile Explorer - Control String

 

With the Parsing method, the user has the ability to add more than one Control String for a given product by repeating the same steps above. The user can remove a Control String for a product by right-clicking and choosing the Edit Control String selection.

 

Barcode Product Sensor (BPS)

WPI systems using the optional BPS and Process Control barcode function will have the ability to activate an alarm if a products barcode label is not read.

 

When the BPS detects a product whose barcode label has not been read, or the BPS is tripped accidentally, an alarm is activated, and the process is stopped via the alarm relay. Alarm 5C appears:

Figure 117: Alarm 5C

 

Control buttons on the screen let the user:

  • Manually reenter the barcode, using the keyboard or a handheld scanner
  • Rescan the barcode with a mounted scanner
  • Continue production without the barcode
  • Remove the board from the line.

Reading Barcodes with a Product Traceability Scanner

When using a Product Traceability Scanner (PTS) to read barcode labels while running VP, the user will get a barcode recorded along with a PWI for every board that passes through the machine. This can be enabled by selecting Traceability under Barcode tab in the Global Preferences.

Use a PTS

When installed on a load conveyor, the PTS reads barcode labels before the products enter the machine. If the barcode label is not read, the process is stopped before the product enters the machine. This method requires an Alarm Relay, and load conveyor.

Search for Barcode Entries

In the Profile Explorer screen, there is a Barcode button located at the bottom of the page. Select this button to search for specific barcoded products and view their profile information for the date/time they were processed.

 

Click on the desired product folder to search, then click on the Barcode button. A dialog box appears with a field to enter the barcode value with a Search button next to it. Enter the value using the keyboard, or by selecting the scanner check box and then using the barcode scanner to scan the actual barcode eliminating the need to enter the value manually.

 

Below this is a field that will list the time, date and PWI for that barcode, if it exists for the chosen product. Either double-click the particular listing or highlight it and then click the View Selected Profile button. This will then bring the user to the Historical Virtual Profile graph for that particular board. If the Charts tab is selected, the Pointer Line will coincide with that particular board for all the charts as well.
