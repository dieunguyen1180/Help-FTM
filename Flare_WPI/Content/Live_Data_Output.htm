Live Data Output

The Live Data Output (LDO) feature outputs Virtual Profile (VP) data in real time for use by third-party applications such as Line Balancing/Production Monitoring Host or SPC software. This data includes barcode information (where applicable) and all relevant profile data, statistics, and limits. There are several different LDO formats that can be chosen from. Examples of these formats will be shown later in this section.

 

Output of the LDO file data – When the VP is running, the statistical data from each newly completed Virtual Profile is written to the LDO file when the board exits the machine.

 

Viewing LDO data –LDO is designed to output data only and is not meant as a historical database. It is intended to be used with third party software that collects the LDO data while the Virtual Profile is running live. If the user isn’t utilizing third party software to collect the LDO data and wish to view historical data, this can be accomplished from within the KIC software in use.

 

LDO destination – The location where the LDO output file is written to can be specified by the user in the ConfigurationProgram.exe utility.

Note: For the Live Data output to function, the appropriately programmed USB dongle key must remain connected to the PC at all times during use. To verify the USB dongle has the LDO feature check in the Software Key panel in the Hardware Status screen.

LDO Formats

 

There are several standard LDO formats to choose from:

 

  • LDO Standard TSV or CSV format for WordPad
  • LDO Standard TSV format for Excel
  • LDO 1 Board 1 File (TXT format)
  • LDO 1 Board 1 File (XML format)
  • LDO 1 Board 1 File (CSV format)

 

 

Details of Output Files

 

TSV and CSV for WordPad

  • The output file type is a .TXT
  • TSV format is Tab delimited
  • CSV format is Comma delimited.
  • The output file is first generated when the VP is started
  • File is appended as each board exits and VP is calculated
  • The default output file name is KIC_WPI_LiveDataOutput.txt
  • The user can choose to define their own specific output file name
  • When default or user defined naming is selected, output file is overwritten when product changeover occurs
  • File name format can also be set to: PRODUCT_YYMMDD_HH-MM-SS.txt
  • When this naming is selected, a new file is generated when product changeover occurs

 

TSV for Excel

  • The output file type is a .TXT
  • TSV format is Tab delimited
  • The output file is first generated when the VP is started
  • File is appended as each board exits and VP is calculated
  • The default output file name is KIC_WPI_LiveDataOutput.txt
  • The user can choose to define their own specific output file name
  • Selecting default/user-defined naming will overwrite the output file during product changeover

 

  • File name format can also be set to: PRODUCT_YYMMDD_HH-MM-SS.txt
  • When this naming is selected, a new file is generated when product changeover occurs

 

One board per file – TXT format

  • The output file type is a .TXT
  • Separate individual file is generated as board exits and VP is calculated
  • File name is ProductName_MachineName_YYMMDD_HH-MM-SS_BarcodeString (Barcode string included when applicable)

  • User cannot change file name – with exception of not including barcode in file name

 

One board per file – XML format

  • The output file type is a .XML file
  • Separate individual file is generated as board exits and VP is calculated
  • File name is ProductName_MachineName_YYMMDD_HH-MM-SS_BarcodeString (Barcode string included when applicable)

  • User cannot change file name – with exception of not including barcode in file name

 

One board per file – CSV format

  • The output file type is a .CSV file
  • Separate individual file is generated as board exits and VP is calculated
  • File name is ProductName_MachineName_YYMMDD_HH-MM-SS_BarcodeString (Barcode string included when applicable)

  • User cannot change file name – with exception of not including barcode in file name

 

Configure LDO

 

  1. With the WPI software shut down, browse to the C:\WPI folder and run the ConfigurationProgram.exe application, and select the LDO tab:

     

    Figure 131: LDO tab - Configuration Program

     

  2. Check the Enable Live Data Output checkbox

  3. Select the desired Format
    Note: The user can view samples of each output type by clicking on the Open button to the left of a selection.

  4. The type of Format chosen will affect what naming structure for the file that will be used (See Details of Output Files above).

  5. Select the destination path for the output file by clicking on the Browse button. This needs to be located on the local drive.

  6. “Max File Size” gives a place to specify how large the LDO output file can get before a new one is started.
    Note: If a VP is running for an extended amount of time, and the Tab or Comma separated for WordPad or Excel is being used, once the maximum file size is reached a new file will be generated and the new file name will be appended with a _# after the file name.

 

Delete Accumulated LDO Files

In time, accumulating LDO files can take up too much hard disk space and need to be deleted. The software provides an Auto Delete function that lets the user specify time intervals for automated file purging. This allows for a specified maximum number of days, up to 1827 (five years), that the system will retain the files before automatic purging. A time can be selected in the 24-hour cycle for the system to automatically check for files that exceed the maximum day limit.

 

  1. In the WPI folder, double-click on the AutoDelete icon.
