Virtual Profiling

Virtual Profiling (VP) is a means of reliably predicting the thermal profile of every board in a production run based on a comparison of real-time temperature and conveyor speed data against a baseline profile established by an earlier profiler run through the machine. For a given product, this baseline needs to represent an optimum machine recipe with a good PWI value and a quality soldering result.

 

During the profiler run, the WPI software simultaneously collects temperature data from the board, from the probes installed at product level, and the other installed sensors. The software calculates a mathematical correlation between these temperature readings and stores this data as part of the baseline profile.

 

Once the baseline is established, the WPI software can be put into inspection mode, recording real-time speed and temperature data as a production run of the same boards passes through the machine. Comparing real-time data to the baseline profile, WPI algorithms accurately extrapolate a simulated thermal profile for each board. WPI updates and saves the VP profile data as each board exits the machine, creating a valuable quality assurance record of each production unit.

 

During inspection mode (VP), the software displays each board’s profile chart and a table of data based on its process window. Other screens show PWI and SPC control charts. When machine data varies significantly from the baseline profile, WPI displays appropriate warnings to system operators.

 

Get a Valid Baseline Profile

Virtual Profiling makes use of a special profile, called the baseline profile, as the data model for its predictions. The software requires that the user first establishes a baseline before it enables Virtual Profiling. The baseline profile should represent the optimum machine recipe for the product and needs to meet the Virtual Profile criteria shown below:

 

  • The PWI must be below the Maximum PWI to allow the VP value as set in Global Preferences.
  • The conveyor speed measured by the encoder must be within 20% of the specified conveyor speed.
  • Probe temperatures must be valid; if any probe thermocouple reports a range of greater than 30C between adjacent readings (every 2.5 seconds), the WPI software will consider them invalid.

 

If the profile meets the above criteria, it can work as a baseline profile. The baseline profile will display differently in the Profile Explorer screen—the profile’s icon will have an overlying green VP.

 

If the user suspects the current baseline profile needs replacement by a newer version that more closely matches current production conditions, the software allows the creation of a new baseline during ongoing production. As long as this verification profile meets VP criteria, it can become the new baseline profile. See Verify The Virtual Profile.

Integrate Empty Machine Data

For a baseline to work as a good predictive model, it needs to use temperature data that reflect both board present and board absent states along the length of the probes. To obtain this data most efficiently, KIC recommends running the profile board with similar boards moving ahead of it and an empty conveyor behind it. The downstream flow of boards stabilizes the temperatures and gives a good approximation of normal production conditions. As the profile board progresses, the probes can measure the temperature when a board is not present. The software can integrate this comparative data for a more accurate predictive capability. The machine should remain empty until the profiler and board completely exits the process chamber.

Note: Be aware of the potential to put the system into a state where the software lacks sufficient empty machine data to make reliable VP predictions. The condition can occur during a verification profile run where the user inserts the profile board into an ongoing production flow. With the other boards both ahead of and behind the profile board, there is no interval for gathering empty machine data. A grey crystal ball indicates WPI’s unable to predict state. See Monitoring Production in Live Mode.

 

Create/Load a Virtual Profile

There are several ways to begin Virtual Profiling:

 

  1. Open Profile Explorer and select the product folder in the upper-left corner. This will display the profiles and events for the selected product. If there’s a profile that meets Virtual Profiling criteria, the profile icon is overlaid with a green VP. This qualifying profile (baseline) will always be the most recent profile that meets Virtual Profiling criteria (only one profile can qualify as the baseline).

    The is no need to highlight the baseline profile, simply click the Start Virtual Profiling button. This will start Virtual profiling for the selected product.

 

  1. If the user has run a profile that meets the Virtual Profile criteria, the Start Virtual Profiling button will appear once that profile has been completed. Click the Start button to begin Virtual Profiling for the indicated product.

 

 

 

  1. If Profile Grouping utility is enabled, click the Link button to associate groups of product models with a baseline profile and to select an individual model for a Virtual Profile production run. When using the Profile Grouping utility, the Start Virtual Profiling button does not appear on the Profile Explorer screen.

Use the Profile Grouping Utility

WPI includes a utility that lets the user identify and group together different products that share the same thermal profile. For example, one might use a physically identical printed circuit board to manufacture many distinct product models. The manufacturer needs to specifically identify and track all the different models, but the thermal process specifications used to solder or cure them are identical throughout the product range. Once a successful process window setup is identified for one model, the Profile Grouping utility lets the user link that setup with other models that can use the same process and thus save the time and cost of entering the same setup specifications repeatedly. In Virtual Profile mode, the baseline profile of a parent product can be matched to multiple child models, permitting smooth production transitions between models.

 

Figure 72: Profile Grouping

 

You enable the Profile Grouping functionality by adjusting settings in the C:\WPI\Log\KIC2000UserSettings.kiccfg file. Once accessing the file, make sure the settings are as shown below:

 

[APPLICATION CONFIGURATIONS]

EnableProfieGrouping=1

Use the Utility with Virtual Profiling

When Profile Grouping is enabled, the user can only start Virtual Profiling by using the utility to identify properly linked product/model pairs. The normal Start button no longer appears on the Profile Explorer screen.
