Profile Explorer

Profile Explorer is a powerful and simple data file management tool. For each unique product name used when profiling, the software will create a folder with the same name. The software saves all the profiles run using that product name in that folder.

 

Figure 64: Profile Explorer

 

The list of product folders is in the upper left of the Profile Explorer.

 

View a Profile:

 

  1. Click on a product folder to display the profiles that have previously been run for any product.
  2. Highlight the profile to view.
  3. Select the Graph button or double-click the profile to display it.

Note: Once the profile appears on screen, the user can use Control + Up Arrow to view the next profile, and Control + Down Arrow to display the previous profile.

The profile section can be sorted by clicking on any of the column headers. Single clicking on a profile will display the Profile description in the upper right corner.

Double-clicking the profile to display the graph and statistics for that profile. Clicking the Display Graph button can also do this.

 

Find VP Production Run Data By Lot ID Code

WPI software can search for Virtual Profile production runs based on the Lot ID code. If the complete code isn’t known, the asterisk symbol (*) can be used as a wildcard to begin searching using only partial code.

 

  1. On the Profile Explorer screen, click on the Lot ID button.

 

The Lot ID search screen appears:

  1. In the Lot ID search screen, type the code in the Enter LOT ID to Search field.

 

  1. Click on the Search button.

 

Note: Matching VP production runs appear listed in the table.

 

  1. Mouse over the row in the table to display pertinent data from the production run in a tooltip popup window:

Figure 66: Lot ID Search Results

 

  1. Highlight the row and click on the History button to display the virtual profile historical data.

 

Browse for Historical Data

The current data path for the data viewed in the Profile Explorer appears in the upper-right corner. This is grayed out if running in production mode. It is active, if the I am not going to run profiles or live WPI button (enabling History mode), is selected when first launching the WPI software.

 

The data path can be changed but must be done through the \WPI\Log\KIC2000DataPath.kiccfg file.

 

To view historical Virtual Profile data from the same PC while Virtual Profiling is live, launch a separate KIC Host.exe file located: C:\WPI\AppForViewerforViewer\KICHost.exe. This will open a separate Profile Explorer that lets the user browse through the history of any product including the one currently running VP. Historical data is available for any boards that have already exited the machine.

 

Note: This feature was designed to allow the user to view WPI historical data stored in a different directory on the same computer or on remote PC over a network.

Profile Explorer Buttons

Lot ID history search – Click on this button to search for a VP production run based on the Lot ID.

Process Traceability Barcode – Click on the Barcode button. A dialog box pops up with a field to enter the barcode value with a Search button next to it. Below this is a field that will list the time, date, and PWI for that barcode if it exists for the chosen product. Either double-click the listing or highlight it and click on the View Selected Profile button.

Start Virtual Profiling – Click this button to start Virtual Profiling. The WPI software will always use the latest qualifying profile as the baseline profile needed to start Virtual Profiling.

Save Selected Profile – Click this button to save an event or profile to the location of the users choice, either hard disk, network drive, or USB flash drive.

Delete – Click this button to delete an event or profile.

Display the Graph and Statistics for this profile – Click this button to display the graph and statistics for the selected profile. If there is an event other than a profile, the Display Graph and Statistics button will change to a Charts button.

Return to KIC Main Menu – Click this button to return to the WPI main screen.

Profile Explorer Event Icons

Profile Ions:

Profile – This icon appears for any valid profile that does not fit Virtual Profile criteria.

 

Virtual Profile capable – This icon appears for the most recent Virtual Profile capable profile. This file will automatically be used as the baseline or reference profile.

 

Virtual Profile Event Icons

Virtual Profile Event Icons

The WPI Profile Explorer will display an event icon for any event that occurs while Virtual Profiling is running and only when Virtual Profiling is running. These events include:

 

VP Start – This icon appears when Virtual profiling is enabled.

 

VP Stop – This icon appears when Virtual Profiliing is disabled.

 

Alarm Icons:

In Spec. – Alarm state change – This icon appears when the Virtual Profiling alarm state changes back to Ready. There are three occasions when this icon may appear:

  • Back into Spec
  • Back into Control
  • Process Temperatures Back to Normal

 

Warning – Alarm state change – This icon appears when the Virtual Profile alarm state changes to Warning. There are three occasions when this icon may appear:

  • CpkWarning
  • Significant Process Temperature Variation
  • Process Is Likely to Go Out of Spec Soon.

 

Out of Spec. – Alarm state change – This icon appears when the Virtual Profile alarm state changes to Out of Spec. There are nine occasions when this icon may appear:

  • Process temperatures are Out of Spec
  • Product length is different from the Virtual Profiling baseline profile
  • Missing barcode (Process Traceability only)
  • The number of barcodes that have not been read. (Process Traceability only)
  • Encoder stopped, or changed signifigantly.
  • DAU/eTPU lost communication

 

Warning Acknowledged – This icon appears once the operator acknowledges the Warning alarm state change.

 

Alarm Acknowledged – TThis icon appears once the operator acknowledges the Out of Spec. alarm state change. Not all alarms require acknowledgement.

Communications Icons:

DAU Communication Error – This icon appears if the software loses communication with the DAU.

 

DAU Communication Restored – This icon appears when the software re-gains communication with the DAU.

 

Process Traceability Barcode failure – This icon appears if the software detects a board in the oven but does not read the barcode for that product.

Rename Profiles

In situations that require it, you have the ability to edit the profile name. This can be especially useful when products are identical but have different names. Once the profile has been optimized, it can be renamed and used as a baseline profile for a different product name if the need arises. This eliminates the need to re-profile products unnecessarily.

Note: Renaming a profile includes the name that is embedded in the profile that shows up on the printout

 

  1. Manually create a new folder in the C:\WPI\Profiles folder using the new product name.

  2. Copy the desired profile into this new folder.

  3. A folder with that new Product Name will appear in Profile Explorer.

  4. When opening that profile, it will display in the software with the new Product Name

  5. The new profile name will appear on all the screens (Profile Explorer, General Tab, Charts Tab, and Troubleshooting Tab). The new name will also appear in the data that is copied to the Clipboard.

  6. None of the files on the PC are actually renamed, when viewing the profiles in Profile Explorer, they all look the same. However, profiles run after the name has been changed will use the new product name as part of their file names.

    Note: Virtual Profiling is most accurate for products that have their own baseline profile.
