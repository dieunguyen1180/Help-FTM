Write Data To and View WPI Data Over A Network

Note: This feature cannot work in conjunction with RPM.

Write Data to a Network Drive

The WPI software can be configured to write the collected data (oven files, process windows, profiles and historical VP data) to a network drive location. This would allow data from one or even multiple systems to be stored in one centralized location (Server/Shared Hard Drive).

 

The first step necessary is to map a network drive from the PC running the WPI software to the desired network location. Due to the multiple varieties of OS and network configurations, KIC cannot detail this step. We recommend the user contact their IT/Network administrator for setting this up. Below is a typical example of this setup.

 

Figure 167: Example Network with WPI

 

Once the drive is mapped, create a folder on the network drive and copy in the following folders from the C:\WPI directory – Ovens, Process Specs, and Profiles. Once completed, follow the steps below on the PC where the WPI software is installed:

 

  1. Ensure the KIC software is closed, open Windows Explorer, and locate the file:
    C:\WPI\Log\KIC2000DataPath.kiccfg

 

  1. The KIC2000DataPath.kiccfg file will open up in Notepad: 

Figure 168

  1. Change the line AllowUserToManuallyChangeWorkingDataPathInThisIniFile=0 to the value =1: 

Figure 169

 

  1. Change the line shown below to = the desired network location

    Note: This will be the main network folder the user created and copied the Log, Ovens, Process Spec, and Profiles folders into.

 

 

Figure 170

 

  1. Once the DataPath.kiccfg file has been modified with the new locations, choose File/Save from the drop-down menu to save the changes.

 

  1. Close Notepad and start the software.

 

  1. With the WPI software open, go to the Profile Explorer:

Figure 171

 

The new network drive should be displayed in the upper right-hand corner of the screen as the Current Data Path.

Note: The Network path will be grayed out and cannot be changed from inside of Profile Explorer.

 

All data collected in this WPI application will now be written to the network folder chosen.

Note: If the user has multiple WPI systems, the user will need to create a separate folder on the Network drive for each system – e.g. KIC Line 1, KIC Line 2, etc.

 

If the network is unavailable when accessing various areas of the software (Define/Edit Process Window, Run A Profile, Profile Explorer), a message will appear alerting the user to the issue:

 

Figure 172

 

When this occurs, the user is still able to use the software but only working with files (ovens, Process Windows, profiles) that are located in the local directory. It will temporarily reset back to the local install directory and any new files generated or updated will be saved in the local directory (C:\WPI). If the network connection is re-established, the WPI software must be restarted to begin writing directly to the network directory again.

 

View Historical Data

The user can also view all collected data over the network from any computer that has the WPI software installed.

 

Use the following steps to view historical data over the network:

 

  1. From a remote computer, start the WPI software and click on the I am not going to run profiles or live WPI button.

    CAUTION: Do not choose this button when working at the PC where the WPI hardware is installed.

 

Figure 173

 

  1. The software will open up normally. When the main screen appears, click the Profile Explorer button.

 

  1. A Browse button is enabled in the upper right-hand corner of the screen:

 

Figure 174

 

  1. You can now click the Browse button and point to the directory where your profiles are.  Keep in mind that you want to direct it to the root directory of the main “Profiles” folder.


For example: If the profiles are in the folder F:\WPI\Profiles\Board A, you would direct it only to the folder F:\WPI\.
