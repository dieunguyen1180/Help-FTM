Remote Process Monitoring

Note: The software key must remain connected at all times during use for RPM capabilities to function. 

 

KKIC Remote Process Monitoring (RPM) enables customers to view real-time thermal process data and process status for any WPI on their wave solder machines from any authorized computer on the network.

 

The WPI system provides valuable thermal process information in real-time. RPM can supply this information to all relevant personaell across the factory. The results are better data management, efficient process management oversight, and faster reaction to thermal process issues in the factory.

 

RPM provides superior information sharing for the factory as a whole. WPI monitors, controls, and displays at the machine, the thermal process information for a single wave solder machine. RPM will, on a single screen, show all the pertinent data for all the machines in the factory on any authorized computer on the network.

 

RPM translates complex process issues into a very easy to understand format. Quality managers, production managers, engineering managers, and line supervisors will, for the first time, access meaningful factory-wide quality and productivity information for the thermal process in real-time without the need for staff meetings, lengthy reports, interpretation, and unnecessary visits to the factory floor

 

RPM is comprised of three main software applications, and one optional software component:

 

  1. WPI software:
    The RPM for the WPI data is a dongle-protected software option. This software runs on the WPI system on a computer at the wave solder machine. RPM’s main function is to send data over a network to the KIC Server software.

    While RPM is enabled, a separate application called WatchDog runs in the background continuously monitoring the connection between each WPI system and the KIC Server(s).

 

  1. KIC Server software:
    The KIC Server software is installed onto a computer or drive on the network and acts as a host for all WPI systems running on the network with the RPM output. It automatically connects to all the WPIs that are running RPM and passes on the data to any authorized computer on the LAN that is properly logged on. A maximum of 100 WPI systems per KIC Server is possible. The KIC Server has a setup menu for creating and editing usernames and passwords for KIC Viewer users.

 

  1. KIC Viewer software:
    The KIC Viewer is installed on any remote computer on the network (i.e. at the users desk) and is used to view the data from all WPI systems connected to the KIC Server. This is done by logging on to the KIC Server.

 

  1. Win VNC:
    Win VNC is a third-party software application that allows for remote control or viewing of the history for any WPI client PC on the same LAN. This can be useful for clearing alarms, loading recipes, and system administration.

 

 

In full operation, the combined software and systems function as follows:

Figure 135: RPM functional diagram

 

  • Each WPI system runs in Virtual Profiling mode, monitoring the process and giving real-time profile/process data for each production board. The Virtual Profile data is stored locally on the WPI client PC.
  • The Machine Name, Product Name, Process Window Index (PWI), Real-Time Cpk, and current Alarm State (green, yellow, red: with alarm text) are sent to the KIC Server as each board exits the machine.
  • The KIC Server accepts the data from all WPIs configured to send data, acting as a host for all the data.
  • The KIC Viewer logs on to the KIC Server and displays the pertinent data for all machines, on one screen, updating each one as boards exit and new data is sent.

O2 Live (Optional)

Note: The software key must remain connected at all times during use for O2 Live capabilities to function.

 

O2 Live is an optional feature that displays an O2 PPM measured value on a profile, or Virtual Profile data. O2 Live is not an oxygen analyzer, but an option that allows for the output from a user’s existing analyzer to be read in and displayed by the WPI software. An external interface device connects between the output of the oxygen analyzer, and the PC (USB port) running the WPI software (See TFS-330210-000 Automatic System Hardware Installation and Maintenance Manual for more details about the interface device). This O2 PPM value is available in the following areas of the software:

 

  • Baseline Profile
  • Virtual Profile (Live)
  • Virtual Profile (Historical data)
  • SPC
  • LDO
  • RPM

 

Configuration

Figure 154: O2 Live Configuration

 

  1. With the WPI software shut down, browse to the C:\WPI folder, run the ConfigurationProgram.exe application and select the O2 PPM tab.
  2. Select Enable O2 PPM Monitor
  3. From the Input Type drop-down menu, choose the option that corresponds with the type of output used on the customer’s oxygen analyzer.
  4. The Low and High values will correspond to the minimum and maximum range of the output; e.g. a Low of 0=0 volts, and a High of 500 equals 5 volts.

  5. Enter the distance from the WPI Board Sensor (at entrance of machine) to the location of the O2 sample port (from the O2 analyzer) inside the machine.

  6. Click the Apply and OK buttons at the bottom of the Configuration Program to save changes and close.

VP Idle Mode (Optional)

Note: The software key must always remain connected during use for VP Idle Mode capabilities to function.

 

VP Idle Mode is an optional feature that allows the WPI software to automatically go into a type of “sleep –mode” during Virtual Profiling if the machine is equipped with an automated ‘energy saving’ capability. When energy savings features are enabled, the machine control software can modify several of the process parameters – such as conveyor speed, or zone setpoint temperatures. When this occurs, the KIC software would normally alarm on the detected changes to the process. Using VP Idle Mode allows the software to ignore those changes during energy saving periods, thus eliminating unnecessary warnings/alarms from being activated.

During VP Idle Mode, the crystal ball will be grey, the statistics chart will be blank, no VP data will be calculated, and no process related alarms or warnings will display or be recorded. When the entrance board sensor is triggered, the system will automatically exit Idle Mode and return to normal VP operation.

Auto-VP

Auto-VP is an available feature that allows for an automated changeover process. Typically, during a product changeover, an Operator has to manually stop the currently running VP, then go into the wave machine software and select and load the correct recipe for the next product to be processed. Then they must go into the KIC software and select the corresponding product and load that VP. This can sometimes lead to the wrong selection being chosen by the user.

 

Through the use of barcodes or MES control programs, the Auto-VP feature can eliminate the need for Operator intervention in the product changeover process. The correct VP is automatically loaded based on information sent to the software.

 

There are multiple configurations of the Auto-VP feature depending on the capabilities and processes in place in the factory. Please contact KIC Tech Support to identify if this functionality can be implemented with the system.
